# paper-analysis

## Purpose
Analyze a paper and extract implementation-relevant information.

## Inputs
- Paper file in PDF, Markdown, or plain text format.

## Outputs
- `paper_summary.md`
- `implementation_spec.md`

## Procedure
1. Identify the paper title and summary.
2. Extract key contributions.
3. Identify algorithms, methods, equations, and pseudocode.
4. Record required input data and output formats.
5. Record required models, libraries, and evaluation methods.
6. Identify missing implementation details.
7. Define MVP scope and excluded scope.

## Rules
- Do not invent paper claims.
- Mark uncertain details as assumptions or unspecified parts.
- For PDF parsing limitations, state the parser boundary clearly.
