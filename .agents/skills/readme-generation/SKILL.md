# readme-generation

## Purpose
Write README documentation for the project or a generated run.

## Inputs
- Project structure
- CLI behavior
- Generated artifacts

## Outputs
- `README.md`
- `README.generated.md`

## Procedure
1. Explain the project purpose.
2. Explain setup.
3. Explain CLI usage.
4. List generated artifacts.
5. Document configuration and limitations.

## Rules
- README must be present before claiming implementation completion.
- Keep setup instructions executable.
