# Evaluation

## Paper Component Coverage
The reviewer lists implemented, partially implemented, and missing paper components. MVP scaffolded items must be marked as partial when they are not executable paper-specific algorithms.

## Algorithmic Fidelity
The reviewer checks whether the paper's algorithmic flow, equations, or pseudocode are reflected in generated artifacts.

## Behavioral Check
The validator confirms required artifacts exist and are non-empty. Future versions should run generated code tests and sample input/output checks.

## Reproducibility
The final report and README must describe execution, setup, dependencies, and configuration.

## Honesty Rule
The system must not claim that a paper-specific algorithm is implemented when the MVP only created a scaffold or plan.
